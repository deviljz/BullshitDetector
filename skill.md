# 谣言检测

## 触发词
- "帮我鉴定这条信息是不是谣言"
- "这是谣言吗"
- "帮我核查"
- "这条消息可信吗"
- "fact check"
- "谣言检测"
- "鉴定谣言"
- "核实一下"
- "这是真的吗"

## 任务定义

对用户提供的信息（文字/URL/图片）进行多步骤事实核查，输出置信度评分、逐点分析和引用来源，并支持多轮追问。

**输入**：文字描述、URL 链接、图片路径（三种均支持）
**输出**：总体结论 + 置信度 + 逐点分析 + 来源列表

---

## 执行流程

### Step 1：输入处理

根据用户提供的内容类型进行预处理：

- **纯文字**：直接进入 Step 2
- **URL**：使用 `ctx_fetch_and_index` 工具抓取页面正文，再进入 Step 2
- **图片路径**：使用 `Read` 工具读取图片（Claude 直接多模态理解），再进入 Step 2
- **混合输入**：分别处理后合并为一段完整描述

### Step 2：声明提取（子 Agent）

派发子 Agent，从内容中识别核心可核查声明。

**子 Agent 指令**：
> 从以下内容中提取 1~5 条核心的、可通过外部证据核查的事实性声明。
> 过滤掉主观意见、情感表达、无法核查的预测。
> 以 JSON 数组格式输出：["声明1", "声明2", ...]
> 内容：{输入内容}

### Step 3：并行搜索（多子 Agent）

对 Step 2 输出的每条声明，**同时派发**独立子 Agent 进行搜索。

**每个子 Agent 指令**：
> 使用 firecrawl_search 工具搜索以下声明的相关证据，尝试 2~3 个不同关键词组合。
> 声明：{声明内容}
>
> 返回格式（JSON）：
> {
>   "claim": "声明原文",
>   "sources": [
>     { "title": "标题", "url": "链接", "snippet": "关键摘要", "stance": "支持/反对/无关", "source_type": "见下方分类" }
>   ]
> }
>
> source_type 分类：
> - "primary"：官方机构、财报、政府数据
> - "independent"：独立媒体/学术机构的原创报道
> - "syndicated"：转载/聚合自其他来源（注明原始来源）
> - "self_reported"：当事方自述（采访、公告、PR稿）
>
> 来源权威性优先级：primary > independent > syndicated > self_reported

### Step 4：证据评估（子 Agent）

汇总所有搜索结果，派发子 Agent 进行评估。

**子 Agent 指令**：
> 根据以下搜索结果，对每条声明进行证据评估。
> {Step 3 全部输出}
>
> 对每条声明输出：
> - 支持来源数 / 反对来源数 / 无关来源数
> - **信源独立性**：这些来源是否真正独立？还是同一份 PR 稿/公告的多次转载？
>   - 若多个来源内容高度雷同、发布时间集中，判定为"同源转载"，只算 1 个有效信源
> - 最权威来源的立场
> - 是否存在明显信息冲突
> - 综合判断，区分以下两种情况：
>   - **"官方自述"**：信息来自当事方，无独立核实，不可标注为"属实"
>   - **"独立核实"**：有 primary 或 independent 来源支持，可标注为"属实"

### Step 4.5：内容性质判断（主 Claude）

在输出报告前，先判断输入内容的性质：

- **宣发性内容**（专访、探访、首曝报道、融资新闻）：在报告开头标注 `⚠️ 原文为宣发/PR性质`，提示用户创始人/制作人自述部分不适合用于事实核查，只核查可独立验证的客观事实。
- **新闻报道**：正常流程。
- **社交媒体/匿名内容**：额外标注来源可信度极低。

### Step 5：裁判输出（主 Claude）

综合 Step 4 的评估结果，生成最终报告，格式如下：

```
━━━━━━━━━━━━━━━━━━━━━━━━
【内容性质】宣发/PR稿 / 新闻报道 / 社交媒体 / 其他

【核查结论】谣言 / 存疑 / 基本属实 / 证据不足

【置信度】XX%
（置信度说明：基于来源数量、独立性、权威性综合评分）

【逐点分析】
▸ 声明1：[独立核实属实 / 官方自述 / 存疑 / 虚假]
  有效信源数：N（注：同源转载只算1个）
  说明：XXX
▸ 声明2：...

【引用来源】
1. [标题](URL) — 类型：primary/independent/syndicated/self_reported
...

【注意】XXX（如信息过新、来源冲突、建议参考专业核查平台等）
━━━━━━━━━━━━━━━━━━━━━━━━
```

## 追问处理

输出报告后，用户可继续追问（如"这个来源可靠吗？""有没有中文来源？""为什么置信度这么低？"）：

- **概念追问**：直接基于已有上下文回答，无需重新搜索
- **新核查需求**：按需重新触发 Step 3（单条声明搜索），不走完整流程
- **来源核实**：对指定来源使用 `ctx_fetch_and_index` 深入抓取核实

---

## 错误处理

| 情况 | 处理方式 |
|------|---------|
| URL 无法抓取 | 提示用户改为粘贴文字内容 |
| 搜索无结果 | 标注"未找到相关来源"，置信度设为"证据不足"，提示可能为新事件或过于小众 |
| 内容为纯主观意见 | 提示该内容属于观点而非事实，无法通过证据核查，建议重新输入具体声明 |
| 来源相互矛盾 | 在逐点分析中明确标注争议，不强行下结论，建议参考专业核查机构 |
| 图片文字无法识别 | 提示用户手动输入图片中的关键文字 |

---

## 局限性说明

执行完成后，在报告末尾附注：

> ⚠️ 本核查基于网络搜索结果，存在以下局限：
> - 极新事件（24小时内）可能搜索不到足够证据
> - 搜索引擎结果不代表全部真相
> - 如需权威核查，建议参考：澎湃新闻核实、Snopes、路透社事实核查等专业平台
